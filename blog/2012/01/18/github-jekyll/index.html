DOM操作においてinnerHTMLは使うべきか(追記)

Jan 18th, 2012

innerHTMLというプロパティがあります。 HTMLを文字列の状態で突っ込むとそのまま反映してくれるという非常に楽で便利なものなんですが、残念なことに、遅い遅いと昔から言われていました。

そうは言っても、document.createElementとかdocument.createTextNodeとかappendChildとか、だるいなあと思いますし、なかなかinnerHTMLの呪縛から逃れられずに早一年。 ちょっとはっきりさせとくかということで、以下のような関数を作り、innerHTMLとどのくらい差が出るか比較してみました。

第一引数に要素名、第二引数に属性と属性値の連想配列、三つ目にはその要素のテキストノードを入れることを想定しています。 ちなみにinnerHTMLも第二引数において使うことができます。 早速自分で書いた関数なのにミスった。 引数は配列で渡します。 第一要素目にエレメント名、第二要素目に属性と属性値の連想配列、第三要素目にエレメントのテキストノードを入れることを想定しています。 ちなみにinnerHTMLも第二要素目において使うことができます。

||IE9|Firefox|GoogleChrome|Safari|Opera| |$1|118|19|12|6|13| |innerHTML|3058|663|2150|939|387|

数値の大小は違えど、innerHTMLよりも速くなりました。 この関数は再帰的に処理しているので、実際にはもっと速くなり得ると思います。 しかし、innerHTMLも内容が小さければ小さいほど速度はあがっていくので、局所的に使う分には問題ないんじゃないでしょうか。
