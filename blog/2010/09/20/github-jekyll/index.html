Sakuraのレンタルサーバにrubygemsをインストールした後に色々インストールしようとしたらハマったので書く

Sep 20th, 2010

タイトル通り、rubygemsインストールして色々やろーって思ってはじめたわけですがハマりました。 あ、それと敬語って慣れないしブログだけ敬語っつーのもなんかアレなのでやめることにします。

さくらのレンタルサーバーに RubyGems をインストールする手順

これを見て進めたんだけど、現状と多少違う点があったので補足的な。 僕自身sshを書くの初めてなんで出来る人からすると「何こいつこんなとこで引っかかってんの？」とか思われるかもしれないけどご了承。

cshrc編集について

主に以下に書かれている点に補足。

% vi ~/.cshrc

set path = (/sbin /bin /usr/sbin /usr/bin /usr/games /usr/local/sbin /usr/local/bin /usr/X11R6/bin)

この行を以下のように変更。

set path = (/sbin /bin /usr/sbin /usr/bin /usr/games /usr/local/sbin /usr/local/bin /usr/X11R6/bin /home/User/bin)

（※/home/User の User はログインユーザー名で置き換える）

それから、RubyGems のインストール先をホームディレクトリ以下に変更したので、環境変数 RUBYLIB と GEM_HOME のパスもこの ~/.cshrc で改めて定義しておく。

setenv RUBYLIB /home/User/lib

setenv GEM_HOME /home/User/lib/ruby/gem

（※/home/User の User はログインユーザー名で置き換える）

これが何をやっているのかは以下の文書に詳しく書いてある。 <<

まず Liquid error: ClassNotFound: no lexer for alias 'sshconfig' found

set pathの中身が大分違った。 実際にはこんな感じ。 Liquid error: ClassNotFound: no lexer for alias 'sshconfig' found

けどまぁサーバですし、毎回同じ環境なわけがないんで 何も気にせず Liquid error: ClassNotFound: no lexer for alias 'sshconfig' found

を付け足して問題ない。(Userはログインユーザー名)

んで、 Liquid error: ClassNotFound: no lexer for alias 'sshconfig' found

を更にcshrcファイルに付け足してviコマンドのZZを叩いて終了。。。

したかに思えた 普通にエラーも出ず処理が終了したんで問題ないと思った。 が！ Liquid error: ClassNotFound: no lexer for alias 'sshconfig' found

どれだけこれを打ち込んでもエラーエラーエラー。 流石になんかおかしいって気づく。

んで、もっかいviコマンドでファイルを書き直したら今度は警告がでた。

Read only file, not written. << はい書き込みできないんだね。何故初回にこれが出なかったのか。 まぁ何でかはわからんし、もしかしたら見落としてたのかも知れんけど とにかく原因がわかったのでftpで変更済cshrcファイルをうｐってEND. Liquid error: ClassNotFound: no lexer for alias 'sshconfig' found

はいおめでとー！

空腹＋無知ってのが無駄に時間を食うはめになってしまった。 上級者ならもっとエレガントにちゃちゃっと終わらせてくれそうです。
