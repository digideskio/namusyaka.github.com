JSにおける配列参照についての正しい解釈？

Sep 30th, 2010

この前僕が書いたエントリーについてid:hirataraさんに色々教えてもらい 更にそれについてのエントリーまで書いていただいたので一応報告。 基本的には僕の前回の言っていることで良いんですが、より掘り下げた解釈も含めてまとめると以下のような感じかと。

  • 配列はオブジェクト。
  • []アクセス演算子の中身は数値であっても内部的に文字列に変換される。
  • また、配列オブジェクトにおけるインデックスは一般的には数値だが、文字列も使用可。
  • 文字列をインデックスに使用した場合は原則的にlengthプロパティは拾わない。
  • 上記の例外として数値と中身が等価な文字列をインデックスに使用した場合はlengthプロパティは拾う。(例):”0” っていうか配列のインデックスは基本この形と思われる。
  • 文字列をインデックスとした配列オブジェクトにおいてforでループさせたりする場合は当然のことながらin演算子を使う。

だけど配列で文字列をインデックスとするくらいならオブジェクトリテラルを用いた連想配列の方が良いんじゃないかと思います。

さくらのレンタルサーバーに RubyGems をインストールする手順

これを見て進めたんだけど、現状と多少違う点があったので補足的な。 僕自身sshを書くの初めてなんで出来る人からすると「何こいつこんなとこで引っかかってんの？」とか思われるかもしれないけどご了承。

cshrc編集について

主に以下に書かれている点に補足。

% vi ~/.cshrc

set path = (/sbin /bin /usr/sbin /usr/bin /usr/games /usr/local/sbin /usr/local/bin /usr/X11R6/bin)

この行を以下のように変更。

set path = (/sbin /bin /usr/sbin /usr/bin /usr/games /usr/local/sbin /usr/local/bin /usr/X11R6/bin /home/User/bin)

（※/home/User の User はログインユーザー名で置き換える）

それから、RubyGems のインストール先をホームディレクトリ以下に変更したので、環境変数 RUBYLIB と GEM_HOME のパスもこの ~/.cshrc で改めて定義しておく。

setenv RUBYLIB /home/User/lib

setenv GEM_HOME /home/User/lib/ruby/gem

（※/home/User の User はログインユーザー名で置き換える）

Twitterに対する考え方

Sep 17th, 2010

Twitterがすごく流行ってます。 実際に使っている人がいるんだから需要があるってことなんでしょう。

だけど

今ひとつコンセプトが明確じゃない気がするんですよね。 ミニブログ　というのに分類されていたりするみたいですが、Twitterにハマる要因っていうのは大抵他の人とのつながりなわけじゃないですか。(それがゆるいかどうかは別にして) それってブログっていうよりかはSNSに近いよなーと。 最近Twitterさん自身もそういう側面を認めるような発言をしたような気がします。

Twitterに固執する理由

つまり何がいいたいか

このご時世色んな素晴らしいサービスが無料で提供されてます。 Twitterに固執せずに要所要所で使い分けることも大事ですよってことです。

新しいサービスを始めたんですよ。 StatsPrinterってやつです。 CSS設定も出来てかなり自由度があると思います。 自己紹介用のパーツとして使ってみるといいんじゃないでしょうか。 こんなかんじで。
